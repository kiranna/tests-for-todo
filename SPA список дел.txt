Функционал:
1.	Добавление записи
2.	Редактирование
3.	Удаление одной записи
4.	Удаление всех записей
5.	Фильтр по полю ID
6.	Фильтр по полю Todo
7.	Фильтр по полю Priority

Проанализировав функционал, можно выделить несколько тест кейсов:

Основные тесты – тесты на базовый функционал.  Тесты в кейсах зависимы друг от друга (это минус, но считаю, что целесообразнее сделать так, чем добавлять каждый раз перед тестами необходимые нам данные. Если бы было предоставлено API, то зависимость можно было бы обойти). К ним относятся тесты:
1.	Тест на проверку открытия страницы
2.	Тест на добавление записи – без упора на проверку вносимых данных
3.	Тест на редактирование добавленной записи
4.	Тест на удаление добавленной записи.
Тесты на важный, но не основной функционал.
5.	Удаление всех записей с подтверждением удаления
6.	Удаление всех записей с отменой удаления
7.	Проверка сортировки по полю ID
8.	Проверка сортировки по полю Todo
9.	Проверка сортировки по полю Priority
Проверка на валидность добавляемых правил. Приведу несколько примеров:
10.	Добавление пустой записи – не заполнив поля
11.	Добавление записи со спец символами
12.	Добавление записи с большим объемом данных
13.	И прочее.

Считаю, обязательны к автоматизации тесты на основной функционал, т.к.он самый критичный. Найдена ошибка в редактировании записи.

После ручного тестирования найдены несколько ошибок в работе функционала общего удаления и сортировки. Поэтому было принято решение тоже их автоматизировать. Но приоритет у них ниже, чем у основных. Не понятен принцип работы сортировки по приоритету, поэтому тест на эту сортировку не пишу, лучше сначала выяснить детали.

Проверка валидности полей возможна лишь частично, т.к. нет ТЗ, описывающего, как именно валидируются поля. При ручном тестировании не было найдено каких-либо ограничений на вводимые данные. В этом случае можно составить предложение по доработке функционала. Но, даже при наличии ограничений, считаю нецелесообразным писать UI тесты на подобные проверки, лучше проверить unit тестами.

С учетом вышенаписанного, автоматизируются тесты на основной функционал, на общее удаление и на сортировку.

Итог: 
написаны тесты на базовый функционал (открытие страницы, добавление записи, редактирование, удаление), тесты на массовое удаление, тесты на сортировку по ID и по названию todo.
Падают тесты:
- редактирование записи
- массовое удаление с отменой удаления
- сортировка по ID (и по возрастанию, и по убыванию).

Затраченное время: 
3 часа на анализ функционала, написание тест плана, описание написанных тестов.
